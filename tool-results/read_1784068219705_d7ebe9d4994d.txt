     1→# ARCHITECTURE & STRUCTURATION CODE — RPG 3D ACTION SCI-FI (UNITY)
     2→
     3→> **Projet :** RPG 3D action sci-fantasy dans l'esprit de **Tower of Fantasy**, **Solo Leveling** (anime) et **Wuthering Waves**.
     4→> **Document de planification technique.** Pense-le comme la feuille de route pour coder un vrai univers persistant 3D, avec ses personnages, ses compétences, son économie, son lore — pas juste un "jeu".
     5→> **Priorité absolue :** le **système** (gameplay, progression, monde vivant) avant le design. Le design s'itera en correctifs après.

     8→
     9→## ⚠️ INSTRUCTIONS POUR L'IA QUI CODE CE PROJET
    10→
    11→> À lire avant toute ligne de code.
    12→
    13→1. **Priorité système > design.** Code d'abord un univers **fonctionnel et complet** (mouvement, combat, IA, progression, économie, quêtes, dialogues, save). Le polish visuel vient après, en correctifs demandés par l'utilisateur.
    14→2. **Demande les assets visuels à l'utilisateur.** Tu ne peux pas coder/modéliser des personnages, environnements, VFX, voix, musique. **Avant de commencer chaque asset visuel**, demande-le à l'utilisateur via une question claire avec un modèle de prompt à lui fournir (voir section "Assets à demander" plus bas).
    15→3. **Style visuel NON-NÉGOCIABLE.** Ne fais jamais un visuel "carton" (low-poly undetailed, couleurs plates criardes). Vise **anime 3D cell-shaded avec dynamisme Solo Leveling** (voir section Style Visuel).
    16→4. **Contrôles tactiles obligatoires.** Le jeu aura des boutons virtuels à l'écran (joystick + 4-6 boutons d'attaque/skill/esquive). Pas seulement clavier/souris.
    17→5. **Pense univers, pas niveau.** Tout doit être data-driven : personnages, compétences, quêtes, régions, dialogues en JSON/ScriptableObjects. On construit un **monde**, pas une suite de scènes.

    75→## 🎨 STYLE VISUEL — NON-NÉGOCIABLE
    76→
    77→### ❌ Ce qu'il ne faut PAS faire (référence : "game type 1.webp")
    78→
    79→> L'utilisateur a explicitement rejeté ce style. À bannir.
    80→
    81→- **Visuel "carton"** : low-poly avec textures plates, pas de détails
    82→- **Couleurs "caillées"** : palette criarde saturée à 100%, sans gradation
    83→- **Animations raides** : pas de tween, pas d'anticipation, pas de follow-through
    84→- **VFX simplistes** : juste des sprites qui poppent, pas de particules dynamiques
    85→- **Lighting plat** : pas de reflets, pas d'ombres douces, pas de bloom
    86→- **Personnages génériques** : sans charisme, sans expression faciale
    87→- **Environnements vides** : sans props, sans végétation, sans atmosphère
    88→
    89→### ✅ Ce qu'il faut viser (référence : "game type 2.webp" — anime 3D Solo Leveling)
    90→
    91→**Philosophie : anime en mouvement, pas jeu vidéo raide.**
    92→
    93→#### Rendu cell-shaded
    94→- **Toon shader** avec ramp shading (3-5 bandes de couleur, pas de gradient lisse)
    95→- **Outlines** épais et noirs sur personnages (style manga)
    96→- **Bloom** sélectif sur éléments lumineux (yeux, armes, VFX)
    97→- **Color grading** : palette saturée mais cohérente (violets, bleus cyan, oranges néon, noirs profonds)
    98→- **Post-processing** : chromatic aberration légère, vignette, film grain subtil
    99→- **Anti-aliasing** : MSAA 4x + FXAA pour edges propres
   100→
   101→#### Dynamisme Solo Leveling
   102→- **Hitstop** : freeze 3-8 frames à chaque hit significatif (jusqu'à 0.1s sur un boss kill)
   103→- **Screen shake** directionnel et dégressif (pas juste un tremblement aléatoire)
   104→- **Slow-motion** : 0.3x pendant 0.5s sur ultimes, parades, dodge parfaits
   105→- **Slash trails** : trails colorés sur armes pendant les attaques
   106→- **Particle bursts** : explosions de particules élémentaires à chaque hit
   107→- **Camera kick** : la caméra recule légèrement à l'impact
   108→- **Camera zoom** : zoom dynamique sur les criticaux / finishes
   109→- **Motion blur** : sur mouvements rapides et attaques
   110→- **Frame freeze** : 1 frame blanche à l'impact (flash hit)
   111→- **Aerial combos** : les ennemis sont liftés en l'air, juggle possible
   112→- **Dramatic angles** : caméra qui tourne autour sur les ultras
   113→
   114→#### Animation
   115→- **Animation rigs** : Humanoid rig avec IK pour pieds/mains
   116→- **Animator** : blend trees pour mouvement (idle/walk/run/jump)
   117→- **Animation events** : déclencheurs VFX/SFX précis sur frames
   118→- **Anticipation** : 3-5 frames de wind-up avant chaque attaque
   119→- **Follow-through** : 5-10 frames de recovery après chaque attaque
   120→- **Squash & stretch** : sur impacts et sauts
   121→- **Facial expressions** : blend shapes pour émotions (joie, colère, surprise, douleur)
   122→- **Lip sync** : synchro bouche sur voix (Oculus LipSync ou custom)

   164→## 🎮 ÉTUDE DE TOWER OF FANTASY + SOLO LEVELING — SYSTÈMES À RÉPLIQUER
   165→
   166→### 1. Système de combat (CRITIQUE — le cœur du jeu)
   167→- **3 armes équipées simultanément** (slot 1/2/3), switch en combat = combo
   168→- **Discharge skill** : jauge de charge 0→1000, libère compétence ultime quand pleine
   169→- **Compétence d'arme** : cooldown court (8-15s)
   170→- **Attaque normale** : combo auto enchaîné (5-7 hits)
   171→- **Charged attack** : maintien 1.5s = attaque lourde
   172→- **Éléments** : Flame / Ice / Volt / Physical / Altered / **Shadow** (Solo Leveling)
   173→- **Résonance élémentaire** : 2 armes même élément → buff (+25% ATK élément, etc.)
   174→- **Résonance d'arme** : 3 armes même type (Tank/DPS/Support/Benediction) → bonus
   175→- **Esquive** : dash avec i-frames, slow-mo si timing parfait (parade)
   176→- **Parade** : blocage juste avant impact = stagger ennemi + contre-attaque
   177→- **Aerial combo** : lift + juggle (style Solo Leveling)
   178→- **Ultimate** : cinématique courte + gros burst damage (0.5s slow-mo)
   179→- **Hitstop + screenshake + VFX** à chaque impact (signature anime)
   180→
   181→### 2. Simulacra (personnages jouables)
   182→- Chaque Simulacrum = skin 3D + arme unique + 2 talents (awakening) + voix
   183→- **Gacha SSR/SR/R/N** — taux 0.75% / 7% / 70% / 22.25%
   184→- **Pity system** : 80 pulls garanti SSR, hard pity 120
   185→- **Duplicates** → fragments d'âme → upgrade talents
   186→- **Awakening** : arbre de talents à débloquer (50+ nodes par perso)
   187→- **Bond system** : affection 1-10, dialogues, skins alt
   188→- **Switch simulacra actif** en combat (change arme + apparence)
   189→
   190→### 3. Équipement & progression
   191→- **Armor** : 7 slots (Head/Hands/Feet/Chest/Legs/Shoulders/Belt) — 4 raretés N/R/SR/SSR
   192→- **Matrices** (chips) : 4 pièces par set, set bonus 2/4 pièces
   193→- **Reliques** : objets activables (jetpack, bouclier, hologramme)
   194→- **Shards de reliques** : farm pour up
   195→- **Enhancement** : +0 → +30 avec matériaux
   196→- **Advancement** : fusion de dupes pour monter étoiles (1→6★)
   197→- **Gear score** : somme de tous les niveaux = power level
   198→- **Skill tree** par simulacrum (50+ nodes)
   199→- **Mastery** : niveaux par élément (Flame Mastery, Ice Mastery, etc.)
   200→
   201→### 4. Open world & exploration
   202→- **Régions** : Astra, Banges, Navia, Crown, Warren, Artificial Island, Miasmic Swamp… (6-10 régions)
   203→- **Streaming** : additive scene loading par chunks (Addressables)
   204→- **Points d'intérêt** : Spatial rifts, supply pods, scenic points, mushrooms puzzles
   205→- **Énigmes environnementales** : activateurs, orbes, surcharges
   206→- **Mounts** : véhicules (motorcycle, mecha, hoverboard) — craft + skins
   207→- **World bosses** : respawn timer, loot partagé multi-joueurs
   208→- **Dungeons instanciés** : raids 4-8 joueurs, world bosses 30 joueurs
   209→- **Weather** : pluie, tempête, day/night cycle, éclairs
   210→- **Verticalité** : saut, vol (jetpack), escalade
   211→
   212→### 5. Multiplayer & social
   213→- **Serveurs partagés** : 1 monde persistant pour N joueurs simultanés
   214→- **Groupes** : 4 joueurs PvE, raid 8
   215→- **Crews** (guilds) : jusqu'à 120 membres, events hebdo, crew battle
   216→- **World chat / privé / crew** + voice chat (option)
   217→- **Trading post** : économie régulée, taxe 5%
   218→- **PvP** : arena 1v1, 3v3, battleground 6v6, open-world PvP flag
   219→- **Spectator mode** : pour esports
   220→
   221→### 6. Monétisation (à transposer éthiquement)
   222→- **Premium currency** : Tanium (payante) + Dark Crystals (farmable)
   223→- **Gacha** : Gold Nucleus / Red Nucleus / Proof of Purchase
   224→- **Battle Pass** : 50 niveaux hebdo
   225→- **Cosmetics** : skins simulacra, mounts, chat bubbles, nameplates
   226→- **Cash shop** : bundles, monthly card, growth fund
   227→- **Limites éthiques** : plafond mensuel, taux affichés, pity counter visible
   228→
   229→### 7. Quêtes & narrative
   230→- **Main quest** (MSQ) — gating par region, 5 acts
   231→- **Side quests** — lore et mats
   232→- **Daily quests** — 5/jour, reset 04:00
   233→- **Weekly quests** — crew + perso
   234→- **Hidden quests** — exploration triggered
   235→- **Events limités** — crossover collabs
   236→- **Dialogue system** (Ink) avec choices, branches, conditions
   237→- **Cutscenes** : Timeline + Cinemachine
   238→- **Voice acting** : FR/EN/JP/CN/KR
   239→
   240→### 8. HUD/UI (signature ToF + Solo Leveling)
   241→- **Top-left** : portrait simulacra + barres HP/charge + buffs
   242→- **Top-right** : minimap + objective tracker
   243→- **Bottom-center** : 3 slots d'armes avec cooldowns + icon skill + ultimate
   244→- **Bottom-right** : inventory quick access + mount button
   245→- **Floating damage numbers** colorés par élément + crit (style Solo Leveling : gros chiffres, pop animations)
   246→- **Combat VFX overlay** : slash trails, elemental bursts, hitstop flash
   247→- **On-screen touch controls** (mobile) : joystick gauche + 6 boutons droite
   248→- **Menus modaux** : Inventory, Simulacra, Crew, Mail, Shop — tous glassmorphism + glitch

   276→### Spécifications techniques
   277→- **Joystick gauche** : flottant ( Dynamic Joystick), 0-1 magnitude, snap-back au release
   278→- **6 boutons droits** :
   279→  1. **Attaque normale** (carré) — combo auto
   280→  2. **Skill d'arme** (triangle) — cooldown 8-15s
   281→  3. **Discharge / Ultimate** (rond, glow quand jauge pleine) — ultime
   282→  4. **Esquive / Dash** (croix) — i-frames
   283→  5. **Swap arme** (L1/R1 style) — switch slot 1↔2↔3
   284→  6. **Inventaire quick access** (carré long press) — radial menu
   285→- **Boutons supplémentaires contextuels** :
   286→  - **Interagir** (triangle quand près d'un PNJ/objet)
   287→  - **Monter véhicule** (quand près d'un mount)
   288→  - **Sauter** (en mode exploration)
   289→- **Gestes tactiles** :
   290→  - **Swipe gauche/droite** : switch arme rapide
   291→  - **Swipe haut** : saut
   292→  - **Swipe bas** : ramasser
   293→  - **Long press** : visée libre (sniper)
   294→- **Haptic feedback** : vibration courte sur hit, longue sur ultime
   295→- **Taille des boutons** : 80-100px (44px min Apple, 48px min Google)
   296→- **Opacité** : 70% au repos, 100% au press
   297→- **Réglages** : taille, opacité, position (gaucher/droitier), sensibilité joystick

   627→## 📋 ÉTAPES DE CODAGE — PLAN COMPLET EN 16 PHASES
   628→
   629→> Chaque phase doit être **validée** (lint + tests + review utilisateur) avant de passer à la suivante. Le système prime sur le design — on construit d'abord un univers fonctionnel, puis on itère le visuel.
   630→
   631→### PHASE 0 — Pré-production (2-4 semaines)
   632→
   633→| Tâche | Détails |
   634→|---|---|
   635→| 0.1 Game Design Document | Document de 50+ pages : vision, pillars, target audience, monétisation éthique |
   636→| 0.2 Style guide visuel | Palette ToF + Solo Leveling, typos, ref artistique, moodboards par région |
   637→| 0.3 Story bible | Lore, factions, personnages, dialogues-clés, 5 acts |
   638→| 0.4 **Demande assets visuels à l'utilisateur** | Voir section "Assets à demander" ci-dessous |
   639→| 0.5 Wireframes UI | Tous les écrans (login, hub, HUD, menus) en Figma |
   640→| 0.6 Prototype graybox | 1 zone test, 1 personnage, 1 ennemi, 1 arme — prouve le game feel |
   641→| 0.7 Validation technique | Build mobile + desktop OK, 60 FPS atteint |
   642→
   643→### PHASE 1 — Socle Unity (1-2 semaines)
   644→
   645→1. Créer projet Unity 6000.0 LTS avec URP
   646→2. Configurer Git LFS pour gros binaires (.fbx, .png, .wav)
   647→3. Installer packages : Input System, Cinemachine, Addressables, TextMeshPro, Odin, DOTween, UniTask
   648→4. Setup dossiers `_Project/` (Core, Gameplay, Data, UI, Network, Render, Audio)
   649→5. **GameManager.cs** singleton + EventBus + ObjectPooler
   650→6. **InputManager.cs** avec Input System (KB/mouse + manette + touch)
   651→7. **SaveSystem.cs** JSON + PlayerPrefs
   652→8. CI GitHub Actions : build auto sur chaque PR
   653→9. **Validation** : build PC + Android + WebGL sans erreur
   654→
   655→### PHASE 2 — Authentification & compte (1 semaine)
   656→
   657→1. Backend **Nakama** en Docker local
   658→2. Inscription email/mot de passe + JWT
   659→3. OAuth2 Google / Apple
   660→4. Schémas DB : `User`, `PlayerProfile`, `Session`
   661→5. UI Login React ou Unity UI Toolkit
   662→6. Validation serveur stricte (schema Zod-like en Go)
   663→7. **Validation** : login + register + session persistante
   664→
   665→### PHASE 3 — Mouvement & caméra (1-2 semaines)
   666→
   667→**Objectif :** Personnage qui court, saute, dash, grimpe dans une scène graybox.
   668→
   669→1. **PlayerController.cs** — CharacterController ou Rigidbody
   670→2. **CameraManager.cs** — Cinemachine virtual camera (3rd person)
   671→3. Animations de mouvement (idle/walk/run/jump/fall) — *demander à l'utilisateur*
   672→4. **InputManager** — bind ZQSD + souris + touch joystick
   673→5. **Dash** avec i-frames + slow-mo sur perfect dodge
   674→6. **Saut** + double saut + wall jump
   675→7. **Grimpe** + ledge grab
   676→8. **Mount system** — base (sans véhicule encore)
   677→9. **Touch controls** — joystick flottant + bouton saut/dash
   678→10. **Validation** : 60 FPS stable, controls responsive < 50ms
   679→
   680→### PHASE 4 — Combat de base (2-3 semaines)
   681→
   682→**Objectif :** Attaquer, prendre des dégâts, tuer 1 ennemi, mourir, respawn.
   683→
   684→1. **PlayerCombat.cs** — attaque normale combo 5 hits
   685→2. **PlayerWeaponManager.cs** — 3 slots + switch animé
   686→3. **EnemyController.cs** + **EnemyAI.cs** — 1 type ennemi basique
   687→4. **DamageCalculator.cs** — formule de base (ATK - DEF)
   688→5. **Health system** — HP, mort, respawn
   689→6. **HitstopController.cs** — freeze 3-5 frames sur hit
   690→7. **ScreenShake.cs** — caméra shake directionnel
   691→8. **FloatingDamage.cs** — chiffres flottants colorés
   692→9. **VFXSpawner.cs** — slash trails, hit particles
   693→10. **Drop & pickup** — loot, magnet, inventory
   694→11. **HUD** — barre HP + compteur score + 3 slots d'armes
   695→12. **Validation** : combat feel bon, hitstop + shake ressentis
   696→
   697→### PHASE 5 — Combat enrichi ToF (3-4 semaines)
   698→
   699→**Réplique du système ToF complet.**
   700→
   701→1. **5 éléments** : Flame / Ice / Volt / Physical / Altered (+ Shadow en bonus SL)
   702→2. **ElementalResolver.cs** — résistances, faiblesses, calculs
   703→3. **WeaponSkill** — cooldown 8-15s par arme
   704→4. **DischargeSystem** — jauge 0→1000, ultimate skill
   705→5. **ComboChain** — enchaînements par arme
   706→6. **ResonanceCalc** :
   707→   - Élémentaire : 2 armes même élément → +25% ATK élément
   708→   - Type arme : 3 DPS → +15% crit, 3 Tank → +20% DEF, etc.
   709→7. **Parade** — blocage timing = stagger + contre
   710→8. **Aerial combo** — lift + juggle
   711→9. **Crit system** — taux + multiplicateur + icône
   712→10. **Buffs/Debuffs** — 50+ types (poison, burn, freeze, stun, ATK up, etc.)
   713→11. **Status effects** — freeze = slow, burn = DoT, etc.
   714→12. **VFX élémentaux** — *demander à l'utilisateur* (flame bursts, ice shards, etc.)
   715→13. **SFX combat** — *demander à l'utilisateur*
   716→14. **Tests balance** — simu 1000 combats, vérif deltas
   717→
   718→### PHASE 6 — Progression & inventaire (3 semaines)
   719→
   720→1. ScriptableObject `ItemData`, `WeaponData`, `ArmorData`, `MatrixData`
   721→2. **PlayerInventory.cs** — 100+ slots, tri, filtre
   722→3. 7 slots d'armor + 3 slots d'arme + 4 slots de matrice
   723→4. **Enhancement** — +0 à +30 avec matériaux (data-driven)
   724→5. **Advancement** — fusion dupes → étoiles 1-6
   725→6. **Gear score** — power level calculé
   726→7. **Matrices** — sets 2/4 pièces, set bonus
   727→8. **Reliques** — 8 slots, activation skill
   728→9. **UI Inventory** complète (filter, sort, compare, lock, dismantle)
   729→10. **Loot tables** — JSON par source (mob, boss, chest)
   730→11. **Anti-cheat** — tout loot server-side, signature cryptographique
   731→
   732→### PHASE 7 — Simulacra & gacha (2-3 semaines)
   733→
   734→1. ScriptableObject `SimulacrumData` (stats, skills, awakening tree, voice lines, lore)
   735→2. 10+ simulacra en data — *demander visuels à l'utilisateur*
   736→3. **GachaService** — RNG seedé (audit), pity 80 / hard pity 120
   737→4. Taux configurables dans `GachaRates.asset`
   738→5. **Fragments d'âme** — dupes converties, awakening
   739→6. UI gacha avec animations (pull card, reveal SSR) — *demander VFX à l'utilisateur*
   740→7. **Éthique** : taux affichés, limite mensuelle, pity counter visible
   741→8. UI Simulacra List (tri, filtre, détails, awakening tree)
   742→9. Switch simulacra actif en combat
   743→10. **Bond system** — affection 1-10, dialogues spéciaux
   744→
   745→### PHASE 8 — Open world & exploration (4-6 semaines)
   746→
   747→1. **WorldStreamer.cs** — additive scene loading par chunks (Addressables)
   748→2. 6 régions ScriptableObjects (Astra, Banges, Navia, Crown, Warren, Artificial Island) — *demander visuels à l'utilisateur*
   749→3. **POIManager** — scenic points, supply pods, rifts
   750→4. **Énigmes** — activateurs, orbes, puzzles mushroom
   751→5. **WeatherSystem** — pluie, tempête, day/night
   752→6. **DayNightCycle** — 24h game = 2h real
   753→7. **World bosses** — scheduling, multi-joueur, loot partagé
   754→8. **Mounts** — 8 véhicules, craft + skins
   755→9. **Minimap** — zoom, POI, ping crew
   756→10. **Fast travel** — unlock scenic points
   757→11. **Hidden quests** — trigger par zone/objet

  1502→## 🎨 CRÉATION DE PERSONNAGE 3D — MÉTHODE CHOISIE
  1503→
  1504→> **Question :** Peut-on créer un vrai personnage 3D pour le jeu depuis une image 2D ?
  1505→> **Réponse :** Oui, via un **pipeline Image-to-3D API**. C'est la meilleure option pour la qualité.
  1506→
  1507→### ❌ Méthodes écartées
  1508→
  1509→| Méthode | Pourquoi écartée |
  1510→|---|---|
  1511→| Mesh procédural C# dans Unity | Blocqueux, qualité type Roblox, pas niveau ToF |
  1512→| Sculpter à la main dans Blender | Je ne peux pas ouvrir Blender/sculpter |
  1513→| Générer un `.psd` | Ce n'est pas un fichier 3D, juste un concept art |
  1514→| Acheter sur asset store | Pas personnalisé au personnage de l'utilisateur |
  1515→
  1516→### ✅ Méthode choisie : Pipeline Image-to-3D (Meshy.ai + Mixamo + Unity)
  1517→
  1518→**Flow complet** (script Python + Unity) :
  1519→
  1520→```
  1521→Image 2D de l'utilisateur
  1522→       ↓
  1523→[1] Script Python : appelle Meshy.ai API
  1524→       → envoie l'image
  1525→       → reçoit un .glb (mesh + textures PBR, ~10k polys)
  1526→       (durée : 1-3 minutes, coût : ~1$ ou gratuit avec quota)
  1527→       ↓
  1528→[2] Auto-rig via Mixamo (semi-manuel, 5 min)
  1529→       → upload du .glb sur mixamo.com
  1530→       → auto-détection des joints
  1531→       → download .fbx riggé Humanoid
  1532→       ↓
  1533→[3] Import dans Unity
  1534→       → .fbx placé dans Assets/_Project/Characters/Player/Models/
  1535→       → Animation Type = Humanoid
  1536→       → Apply
  1537→       ↓
  1538→[4] Application du Toon Shader URP (déjà codé)
  1539→       → StellarOdyssey/Toon shader appliqué
  1540→       → Cell-shading + outlines noirs manga
  1541→       → Palette personnalisée (noir/cyan/bleu)
  1542→       ↓
  1543→[5] Animations procédurales (déjà codées dans PlayerAnimator3D.cs)
  1544→       → idle, walk, run, jump, attack combos, dash, hit reaction
  1545→       → Pas besoin de fichiers .anim externes
  1546→       ↓
  1547→[6] Prefab Unity assemblé via config JSON
  1548→       → CharacterGenerator + PlayerController3D + PlayerAnimator3D + PlayerCombat3D
  1549→       → Personnage prêt à jouer
  1550→```
  1551→
  1552→### Fichiers fournis
  1553→
  1554→| Fichier | Rôle |
  1555→|---|---|
  1556→| `unity-project/tools/generate_character_3d.py` | Script Python qui pilote Meshy.ai API + génère config Unity |
  1557→| `unity-project/Assets/_Project/Characters/Player/CharacterGenerator.cs` | Générateur de personnage procédural (fallback si pas d'API) |
  1558→| `unity-project/Assets/_Project/Characters/Player/PlayerAnimator3D.cs` | Animations procédurales dynamiques |
  1559→| `unity-project/Assets/_Project/Characters/Player/PlayerController3D.cs` | Contrôleur mouvement 3rd person |
  1560→| `unity-project/Assets/_Project/Characters/Player/PlayerCombat3D.cs` | Système de combat (combos + hitstop + VFX) |
  1561→| `unity-project/Assets/_Project/Rendering/ToonShader.shader` | Shader URP cell-shaded + outlines |

  1565→```bash

  1761→├── 2. STUNNED → wait (status: stun/freeze)
  1762→├── 3. FLEE → if fear > 0.7 AND anger < 0.8
  1763→│   └── Flee player + Seek spawn + CallForHelp
  1764→├── 4. BERSERK → if anger > 0.8
  1765→│   └── Charge player + Attack (ignore fear)
  1766→├── 5. INVESTIGATE → if heard/saw but lost player
  1767→│   └── Move to lastKnownPos + search pattern
  1768→├── 6. ATTACK → if can see + in range + cooldown ok + slot available
  1769→│   └── Face player + Deal damage + Release slot
  1770→├── 7. CHASE → if can see player, not in range
  1771→│   └── Flank position + Arrive + ObstacleAvoid
  1772→├── 8. RETREAT → if too far from spawn (> leashDistance)
  1773→│   └── Arrive spawn
  1774→└── 9. PATROL (default)
  1775→    └── Wander + ObstacleAvoid + Return to spawn if far
  1776→```
